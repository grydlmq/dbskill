# dbskill 新手入门

你可能是看到有人用 dbskill 梳理商业和内容问题，想知道自己该怎么开始。

dbskill 由自媒体博主 dontbesilent 创建，全网可搜 `@dontbesilent 聊赚钱`。dontbesilent 累计发布 16152 条推文，dbskill 将其中经过筛选的内容整理为 4176 个知识原子和 26 个 AI Skills，帮助你处理商业判断、内容创作、个人行动与长期决策。

你不需要记住工具名称，也不需要先把问题说得很专业。先把真实处境交出来，dbskill 会帮你找到切入点。

## 第一次怎么用

第一次使用，直接输入：

```text
/dbs 新手入门
```

dbskill 会先了解你正在做什么，一次只问一个问题，再推荐合适的 Skill，并直接带你完成第一次实际使用。

你也可以输入 `/dbs`，然后补全下面任意一句：

```text
我在做【什么】，现在卡在【哪里】。
我想通过内容获得【什么结果】，但不知道该做什么。
我看到【谁／什么案例】做得很好，想判断自己该不该学。
我知道自己该做【什么】，但一直没开始，因为【什么】。
```

例如：

```text
/dbs 我做少儿编程课，已经有 40 个付费学员，但续费低。我不知道是产品问题、定价问题，还是我找错了人。
```

`/dbs` 会读当前对话里已经给出的信息，直接把你带到合适的 Skill。你无需重复描述背景。

## 已经知道要做什么

熟悉 dbskill 后，可以跳过 `/dbs`，直接调用具体 Skill。命令后面直接写材料或问题：

```text
/dbs-diagnosis 我想卖一门给自由职业者的 AI 课，怎样判断这门生意是否成立？
/dbs-hook 这是我短视频的前 20 秒，帮我优化开头：……
/dbs-benchmark 我想研究做企业服务内容的账号，帮我找对标。
```

每个 Skill 都会继续追问真正缺少的信息。已经出现在当前对话里的信息，不需要重新回答。

## 选择一个起点

| 你现在的处境 | 直接调用 |
|---|---|
| 有业务、产品或赚钱问题，想判断方向 | `/dbs-diagnosis` |
| 想做内容，想知道选题或文稿怎么做 | `/dbs-content` |
| 想找学习对象，判断该模仿谁 | `/dbs-benchmark` |
| 知道该做什么，却迟迟没有行动 | `/dbs-action` |
| 连问题或目标本身都说不清 | `/dbs-good-question` 或 `/dbs-goal` |
| 只想讨论一个问题，听不同视角 | `/dbs-chatroom` |

## Skill 全目录

以下列出当前正式发布的 26 个 Skill。每项均包含适用时机、可直接输入的话与常见衔接方式。名称带 `beta` 的本地试验 Skill 不在本页范围内。

### 入口与商业判断

#### `/dbs`｜主入口与下一步导航

**什么时候用：** 你还不确定该从哪里开始，或刚完成一轮 dbskill 工作，想知道下一步。

**你可以这样说：** `/dbs 我有一个线上训练营想法，但不知道该先验证什么。`

**你会得到：** 基于当前对话的直接路由；完成某个 Skill 后，会结合结论推荐 2 至 3 个后续方向。

**常见衔接：** `/dbs-diagnosis`、`/dbs-content`、`/dbs-benchmark`。

#### `/dbs-diagnosis`｜商业模式诊断

**什么时候用：** 你要判断一门生意、产品、定价、客户或方向是否成立；也适合对已有业务做系统体检。

**你可以这样说：** `/dbs-diagnosis 我做面向宝妈的收纳咨询，客户总觉得贵。我该降价还是换交付？`

**你会得到：** 问题的消解过程，或一份包含核心判断、风险和处方的商业模式诊断报告。

**常见衔接：** 想找参考路径用 `/dbs-benchmark`；商业方向定下后用 `/dbs-content`；长期追踪验证用 `/dbs-decision`。

#### `/dbs-benchmark`｜对标分析

**什么时候用：** 你想找值得长期研究的对象，或已经有候选对象，想判断是否值得学。

**你可以这样说：** `/dbs-benchmark 我想做面向中小老板的短视频，应该研究哪些账号？`

**你会得到：** 经五重过滤后的对标建议，以及具体该研究什么、怎样避免把个人偏好误当判断。

**常见衔接：** 对标确定后用 `/dbs-content` 把学习转成内容行动；持续记录模仿结果用 `/dbs-decision`。

#### `/dbs-slowisfast`｜慢就是快

**什么时候用：** 你在关键选择上一直想找捷径，或需要判断哪件事值得长期投入、反复打磨。

**你可以这样说：** `/dbs-slowisfast 我想快速起号，每天追热点有用吗？`

**你会得到：** 当前方法的检测、值得慢做的环节、无需慢做的环节和下一步动作。

**常见衔接：** 不知道该研究谁用 `/dbs-benchmark`；方向本身有疑问用 `/dbs-diagnosis`；知道答案却做不动用 `/dbs-action`。

#### `/dbs-deconstruct`｜概念拆解

**什么时候用：** 一个词在不断影响你的判断，但你说不清它的含义，例如「个人 IP」「赛道」「影响力」「创业」。

**你可以这样说：** `/dbs-deconstruct 个人 IP 到底是什么？`

**你会得到：** 概念在不同场景中的用法、可检验的还原和它与当前问题的关系。

**常见衔接：** 概念背后是生意问题时用 `/dbs-diagnosis`；整句话其实是模糊目标时用 `/dbs-goal`。

#### `/dbs-goal`｜目标清晰化

**什么时候用：** 你说「想做个人 IP」「想更有影响力」「想变得更好」，却无法确认要完成什么。

**你可以这样说：** `/dbs-goal 我想成为一个有影响力的商业博主。`

**你会得到：** 原目标的语义审计、可检查的重写版、验收条件和第一步行动。

**常见衔接：** 目标清楚后行动受阻用 `/dbs-action`；目标牵涉商业选择用 `/dbs-diagnosis`；需要长期回填用 `/dbs-decision`。

#### `/dbs-good-question`｜好问题生成器

**什么时候用：** 你有困惑，却无法把它表达成 AI 或他人能真正处理的问题；也适合判断一个问题可自动化到什么程度。

**你可以这样说：** `/dbs-good-question 我总觉得内容不对，但说不出具体哪里不对。`

**你会得到：** 问题的断点、需要补充的事实、可推理的问题说明书，以及 Agent 可解性的判断。

**常见衔接：** 问题涉及商业模式用 `/dbs-diagnosis`；涉及内容表现用 `/dbs-content` 或 `/dbs-hook`；涉及对标选择用 `/dbs-benchmark`。

### 内容创作与传播

#### `/dbs-content`｜内容创作诊断

**什么时候用：** 选题已经有了，想判断怎样把它做成内容，或想诊断一篇文案、一条视频应该怎样调整。

**你可以这样说：** `/dbs-content 我想讲“普通人别急着做个人 IP”，这个选题怎么做？`

**你会得到：** 推荐形式、五维诊断、具体制作方向、第一步行动和必要的风险提醒。

**常见衔接：** 开头专项优化用 `/dbs-hook`；标题用 `/dbs-xhs-title`；定稿后检查共鸣用 `/dbs-resonate`。

#### `/dbs-hook`｜短视频开头优化

**什么时候用：** 你已有短视频开头，想知道它为什么留不住人，并获得多套可用改法。

**你可以这样说：** `/dbs-hook 我的开头是“今天聊聊为什么做内容这么难”，帮我优化。`

**你会得到：** 当前开头诊断、多个优化方案和 Top 3 推荐。

**常见衔接：** 开头确定后用 `/dbs-script-flow` 检查全稿衔接；发现选题本身有问题时回到 `/dbs-content`。

#### `/dbs-xhs-title`｜小红书标题公式

**什么时候用：** 你有明确的内容主题，需要为小红书生成标题并理解为什么这样写。

**你可以这样说：** `/dbs-xhs-title 主题是“自由职业者怎样报价”，行业是咨询。`

**你会得到：** 匹配话题的标题方案、Top 3 推荐与对应公式说明。

**常见衔接：** 标题之后做开头用 `/dbs-hook`；要检查整篇内容用 `/dbs-content`。

#### `/dbs-resonate`｜文稿共鸣诊断

**什么时候用：** 文稿已经写完，担心没有击中人、完播率低或发出去没有反应。

**你可以这样说：** `/dbs-resonate 这是我的逐字稿，帮我看它有没有戳中真正的受众：……`

**你会得到：** 共鸣强弱判断、情绪与立场的断点、具体删改建议。

**常见衔接：** 内容方向仍不清楚用 `/dbs-content`；开头套路感太强用 `/dbs-hook`；逐段衔接有问题用 `/dbs-script-flow`。

#### `/dbs-spread`｜传播心理解码

**什么时候用：** 你想理解一段已有内容为什么能火、击中了什么情绪，或受众接下来还想听什么。

**你可以这样说：** `/dbs-spread 这条内容为什么引发这么多转发？内容是：……`

**你会得到：** 基于 5 个传播学理论的共鸣机制、有效立场、受众情绪和可继续讨论的方向。

**常见衔接：** 有了传播角度后用 `/dbs-xhs-title`；想做多视角发散用 `/dbs-chatroom`；把方向变成成品用 `/dbs-content`。

#### `/dbs-script-flow`｜逻辑延续检查

**什么时候用：** 你有短视频逐字稿，想找出段落衔接、信息密度或口播流畅度造成的划走风险。

**你可以这样说：** `/dbs-script-flow 帮我检查这篇逐字稿哪里会让观众划走：……`

**你会得到：** 分段后的三维检查报告；确认后还可以做保留原文痕迹的标记式改稿。

**常见衔接：** 开头薄弱用 `/dbs-hook`；整体方向错误用 `/dbs-content`；发出前担心共鸣用 `/dbs-resonate`。

#### `/dbs-ai-check`｜AI 写作特征识别

**什么时候用：** 你想检查文案有没有明显的 AI 腔、套路化结构或失真的表达。

**你可以这样说：** `/dbs-ai-check 帮我检查这篇文案有没有 AI 味：……`

**你会得到：** AI 特征检测报告。默认只诊断；你明确说要改时，才进入改写引导。

**常见衔接：** 开头有套路感用 `/dbs-hook`；选题和表达方向都有问题用 `/dbs-content`。

#### `/dbs-wechat-html`｜微信公众号 HTML 生成

**什么时候用：** 文章已经以 Markdown 写好，需要生成可粘贴到微信公众号后台的排版 HTML。

**你可以这样说：** `/dbs-wechat-html 我的文章.md，做成适合商业分析的风格。`

**你会得到：** 1 个指定或推荐风格、6 个推荐风格，或全部 15 个风格的 HTML 与预览页。

**常见衔接：** 内容定稿前用 `/dbs-content` 或 `/dbs-resonate`；完成排版后直接发布。

#### `/dbs-content-system`｜内容结构化系统

**什么时候用：** 你已有大量本地文稿、推文、选题、案例或课件，想把它们变成可检索、可重组、可持续生长的内容资产。

**你可以这样说：** `/dbs-content-system 我有 800 篇公众号文章和课程稿，想搭一个内容资产工程。`

**你会得到：** 素材审计、工程结构、内容单元、主题地图与选题装配稿。

**常见衔接：** 资产系统建好后，用 `/dbs-content` 诊断具体选题；有长期规律需要回填时用 `/dbs-decision`。

### 行动、学习与长期记录

#### `/dbs-action`｜执行力诊断

**什么时候用：** 你知道该做什么，却总在拖延、准备、反复换方向或回避行动。

**你可以这样说：** `/dbs-action 我知道该每天发视频，但每次写到一半就去研究别人的账号。`

**你会得到：** 对行动阻塞信号的诊断、原因判断和能实际开始的下一步。

**常见衔接：** 方向还没确定用 `/dbs-diagnosis` 或 `/dbs-benchmark`；目标本身不清楚用 `/dbs-goal`；方法总在找捷径用 `/dbs-slowisfast`。

#### `/dbs-learning`｜交互式学习

**什么时候用：** 你想系统学习一个主题，希望 AI 根据每一篇后的真实反馈调整下一篇。

**你可以这样说：** `/dbs-learning 带我系统学习定价，每篇都留一道让我反馈的问题。`

**你会得到：** 一份本地学习计划和连续文章；后续输入「继续下一篇」即可基于反馈推进。

**常见衔接：** 学习中发现概念不清用 `/dbs-deconstruct`；想把理论应用到具体商业问题用 `/dbs-diagnosis`。

#### `/dbs-decision`｜个人决策系统

**什么时候用：** 你需要长期追踪业务、关系、健康、职业、学习或投资中的判断，并通过结果回填逐渐发现规律。

**你可以这样说：** `/dbs-decision 我想长期记录自己的内容业务决策，后面能回看哪些判断有效。`

**你会得到：** 本地决策工程、事件立案、结果回填、状态画像和可升级的规律库。

**常见衔接：** 判断方向是否成立用 `/dbs-diagnosis`；找参考路径用 `/dbs-benchmark`；目标和问题模糊时先用 `/dbs-goal` 或 `/dbs-good-question`。

#### `/dbs-save`｜诊断存档

**什么时候用：** 当前对话已经得出关键结论，想在下次继续，而不从头讲背景。

**你可以这样说：** `/dbs-save` 或 `/dbs-save 训练营定价判断`

**你会得到：** 当前项目下的一份结构化存档，记录结论、否决方向、待验证假设与推荐下一步。

**常见衔接：** 下次回来用 `/dbs-restore`；积累多份存档后用 `/dbs-report`。

#### `/dbs-restore`｜接续诊断

**什么时候用：** 你想把上次保存的诊断状态拉回来，继续处理。

**你可以这样说：** `/dbs-restore`、`/dbs-restore list` 或「接着上次」。

**你会得到：** 当前项目最近一次存档及其中已确定的结论、未解决问题和推荐下一步。

**常见衔接：** 根据存档中的 `next_skill` 继续对应工作；不知道选哪条时输入 `/dbs`。

#### `/dbs-report`｜诊断报告

**什么时候用：** 你已积累多次诊断存档，想合并成一份可分享给合伙人、客户或自己复盘的 Markdown 报告。

**你可以这样说：** `/dbs-report` 或 `/dbs-report --since 2026-07-01`

**你会得到：** 按时间整理、去重并分类后的报告文件。

**常见衔接：** 报告中有未解决问题时，回到对应的诊断 Skill；新结论需要继续追踪时用 `/dbs-decision`。

### 多视角讨论

#### `/dbs-chatroom`｜定向聊天室

**什么时候用：** 你希望从多个高质量视角讨论一个问题，或想指定人物参与对话。

**你可以这样说：** `/dbs-chatroom 我想讨论“内容创作者该不该做付费社群”，请帮我找几种互相冲突的视角。`

**你会得到：** 与问题匹配的专家视角、多角色讨论和最后的判官总结。

**常见衔接：** 讨论出明确商业问题用 `/dbs-diagnosis`；形成内容方向用 `/dbs-content`。

#### `/dbs-chatroom-austrian`｜奥派经济聊天室

**什么时候用：** 问题涉及价格、利润、市场、企业家精神或创业与做生意的边界，想从奥派经济学深入讨论。

**你可以这样说：** `/dbs-chatroom-austrian 为什么很多人把创业和做生意混为一谈？`

**你会得到：** 哈耶克、米塞斯与 Claude 的多角色讨论，以及可继续检验的结论。

**常见衔接：** 讨论落到具体业务时用 `/dbs-diagnosis`；要细拆一个概念时用 `/dbs-deconstruct`。

### Agent 工作台

#### `/dbs-agent-migration`｜Agent 工作台迁移

**什么时候用：** 你在 Claude Code、Codex、Grok、豆包或 Trae 等多个 Agent 之间使用项目，规则文件和 Skill 逐渐混乱，需要建立可长期维护的统一工作台。

**你可以这样说：** `/dbs-agent-migration 我想把这个项目统一到 Claude Code、Codex 和 Grok。`

**你会得到：** 规则文件审计、真源识别、命名整理和多端 bridge 方案。

**常见衔接：** 只想把某个已有 Skill 挂到多个 Agent 时，直接用 `/dbs-bridge`。

#### `/dbs-bridge`｜多端 Skill 桥接

**什么时候用：** 你已有一个 Skill 或一个 `skills/` 目录，想让 Claude Code、Codex、通用 Agents 与 Grok 都能调用，并保持单一真源。

**你可以这样说：** `/dbs-bridge skills/dbs-hook` 或 `/dbs-bridge skills`

**你会得到：** 软链 bridge 与 Grok 薄 bridge；也支持查看桥接状态和取消桥接。

**常见衔接：** 工作台整体存在重复规则、旧文件或命名混乱时，先用 `/dbs-agent-migration`。

## 常用组合路径

组合按你要完成的事选择。每一步都会根据前一步结论调整，路径不是机械流程。

### 1. 从零开始做内容

```text
/dbs
  → /dbs-content
  → /dbs-hook
  → /dbs-xhs-title
  → /dbs-script-flow 或 /dbs-resonate
```

先用 `/dbs` 说清楚自己的业务、受众与目标；`/dbs-content` 确认内容方向；`/dbs-hook` 处理前几秒；`/dbs-xhs-title` 生成标题；定稿后按需要检查逻辑衔接或共鸣。

### 2. 判断一个生意值不值得做

```text
/dbs-diagnosis
  → /dbs-benchmark
  → /dbs-decision
  → /dbs-save
```

先拆清楚生意和问题；再找能提供现实参照的对标；把关键选择与结果持续回填；每次阶段性结论用存档保留。

### 3. 从爆款中提炼下一条内容

```text
/dbs-spread
  → /dbs-chatroom
  → /dbs-content
  → /dbs-hook
```

先分析一段内容击中的心理与立场；需要时用多视角讨论扩展；再把方向变成你的内容，最后处理开头。

### 4. 写完一篇内容到发布

```text
/dbs-resonate
  → /dbs-script-flow
  → /dbs-ai-check
  → /dbs-wechat-html
```

先检查内容是否真正击中人；再处理逐段衔接；确认文字没有明显 AI 痕迹；最后生成微信公众号 HTML。短视频可在前两步之间加入 `/dbs-hook`。

### 5. 知道该做却一直做不动

```text
/dbs-action
  → /dbs-goal 或 /dbs-diagnosis
  → /dbs-slowisfast
```

先找行动卡住的具体信号；目标空泛时审计目标，方向不清时诊断商业问题；若持续在找捷径，再检查哪些摩擦值得长期承担。

### 6. 建立自己的内容资产与决策记忆

```text
/dbs-content-system
  → /dbs-content
  → /dbs-decision
  → /dbs-save
  → /dbs-report
```

先把旧文稿、案例和课程变成结构化资产；用它继续生产和诊断内容；将关键判断、验证结果与阶段结论持续沉淀。

## 使用习惯

- **材料越真实，结果越具体。** 说清楚你卖什么、面对谁、已经试过什么、哪里失败了。
- **不用重复背景。** `/dbs` 和各个 Skill 应先利用当前对话里的信息；只补充真正缺少的事实。
- **跨对话请存档。** 当前聊天记录关闭后，输入 `/dbs-restore` 拉回上次结论。
- **不知道下一步时回到 `/dbs`。** 它会按当前结论推荐值得继续走的方向。
